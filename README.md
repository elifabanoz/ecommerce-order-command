# ecommerce-order-command
Command Pattern implementation for e-commerce order management system. Handles order processing and cancellation with undo functionality.

Case: 
Suppose that you are working for an e-commerce company selling products and goods. The customers add
the items to the chart and do payments in the system to buy products. After that, the order is processed to
the logistics. To deal with the order cancellation efficiently after the order is processed, which design pattern
should be implemented?
